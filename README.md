# IGNA Competitive Product Research Agent

An intelligent, category-agnostic product research agent designed to seamlessly scrape, analyze, and report e-commerce data across major storefronts (Amazon, Best Buy, and eBay). 

Built with **FastAPI**, **Playwright**, and **Azure OpenAI**, IGNA automates the process of gathering competitive pricing, product details, and market insights.

## 🚀 Key Features

* **Multi-Store Web Scraping:** Simultaneously extracts product data from Amazon, Best Buy, and eBay.
* **Resilient Playwright Automation:** Includes CAPTCHA-bypass logic, bot-evasion tactics, header spoofing, and robust DOM parsing (handles comma-formatted prices, hidden elements, etc.).
* **LLM-Powered Intelligence (`igna_brain`):** Uses Azure OpenAI to interpret user requests, extract search intent, and summarize the final market data.
* **Automated Reporting:** Outputs cleaned, structured data into JSON and CSV formats automatically timestamped in the `data/` directory.
* **Modern Asynchronous Backend:** Built on FastAPI with dedicated thread-pooling (`asyncio.to_thread`) to safely run Playwright's EventLoop without blocking the API under Python.

## 🧠 System Architecture & API Flow

The system flows from a single API request down to multi-threaded web scrapers, utilizing the LLM for query understanding.

### Step-by-Step API Flow

1. **Client Request:** 
   The user sends an HTTP POST request to the API containing a natural language query (e.g., *"Find the best prices for a Sony A7IV camera"*).
2. **Intent Parsing (`igna_brain.py`):** 
   The request is passed to the AI brain (powered by Azure OpenAI). It translates the natural language into precise search parameters and validation rules.
3. **Scraper Dispatch (`api.py`):** 
   FastAPI triggers the background scrapers via `asyncio.to_thread()`. This executes the Playwright browser automation in a separate thread, avoiding `ASGI` event loop conflicts.
4. **Data Extraction (`cua_scraper.py`):** 
   Playwright opens headless browsers and executes targeted search queries directly on Amazon, Best Buy, and eBay. It sanitizes prices (e.g., converts `"$1,149.99"` to `1149.99`) and retrieves titles, URls, and ratings.
5. **Data Structuring & Validation (`models.py`):** 
   Raw scraped data is normalized into Pydantic models to ensure strict schema adherence (preventing 500/422 errors).
6. **Report Generation (`reporter.py`):** 
   The validated records are written to disk in the `data/` folder as both `.csv` and `.json` artifacts.
7. **Client Response:** 
   Finally, the API returns a consolidated 200 OK JSON response containing the structured competitive analysis back to the original client.

## ⚙️ Getting Started

### Prerequisites
* Python 3.9+ (Tested on 3.14 on Windows)
* An active Azure OpenAI subscription.

### Installation

1. **Clone the repository:**
   ```bash
   git clone https://github.com/priyanshishukla-vibecoder/igna_agent_poc.git
   cd igna_agent_poc
   ```

2. **Install dependencies:**
   ```bash
   pip install -r requirements.txt
   playwright install chromium
   ```

3. **Configure Environment:**
   Create a `.env` file in the root directory and add your Azure OpenAI keys:
   ```env
   AZURE_OPENAI_API_KEY=your_key_here
   AZURE_OPENAI_ENDPOINT=your_endpoint_here
   ```

4. **Run the API Server:**
   ```bash
   uvicorn api:app --reload
   ```

## 🔧 Troubleshooting

* **EventLoop Errors on Windows:** Handled natively in the code. We utilize `ProactorEventLoop` isolated in separate threads for Playwright to ensure compatibility with ASGI servers like Uvicorn.
* **Amazon Bot Blocks:** The scraper uses heavy HTTP Header configuration and automatic retry hooks to bypass initial CAPTCHA screens.
